Self-hosted AI agent platform built on top of pi-coding-agent. Run agents on your server with multi-user Linux isolation. Web UI for chat, files, terminal, and memory. Bring any LLM provider including local ones.

oqto is in early, active development. expect bugs and unfinished features. llm agents are susceptible to prompt injection and data exfiltration. use sandboxing and scoped permissions, and avoid giving agents unrestricted access to sensitive data.

git clone https://github.com/byteowlz/oqto.git && cd oqto && ./setup.sh
Or use the comfortable configurator: oqto setup configurator

features

Chat + Canvas

Streaming responses, tool calls, file refs, multi-modal input.

Terminal + Files

Full terminal emulation and syntax-highlighted file browser in the browser.

Memory

Semantic search across conversations via mmry.

Multi-User

Per-user Linux isolation with dedicated UIDs and namespace sandboxing.

Dual Runtime

Local mode via runner daemon or container mode with Docker/Podman.

Sandbox

Layered config: global policy + per-workspace restrictions. Workspaces can only tighten.
