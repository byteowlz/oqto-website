architecture

clean separation of concerns

Canonical Protocol

The frontend speaks a harness-agnostic protocol over multiplexed WebSocket channels. Runtime agnostic by design -- swap agents without touching the UI.

Runner Isolation

Backend never spawns agents directly. All processes go through octo-runner with per-user Linux isolation, namespace sandboxing, and filesystem restrictions.

Dual Runtime

Local mode via runner daemon for single or multi-user Linux, or container mode with Docker/Podman for full isolation. Even local mode uses the runner indirection.

security

defense in depth by default

Process Isolation

Every agent process runs through octo-runner, never spawned by the backend directly. In multi-user mode each user maps to a dedicated Linux user (octo_{username}) with separate UID/GID.

Namespace Sandboxing

Agents run inside bubblewrap (bwrap) namespaces on Linux or sandbox-exec Seatbelt profiles on macOS. Mount, PID, and network isolation with fine-grained filesystem ACLs.

Layered Sandbox Config

Global admin restrictions in config.toml combined with per-workspace .octo/sandbox.toml. Workspaces can only ADD restrictions, never weaken global policy.

[local.sandbox]
enabled = true
profile = "development"
deny_read  = ["~/.ssh", "~/.gnupg"]
deny_write = ["/etc", "/usr"]
isolate_network = false
isolate_pid = true

JWT Authentication

All API access requires JWT tokens with invite-code-based registration. Multi-user deployments enforce minimum 32-character secrets with role-based access control.

Sudoers Hardening

Multi-user mode uses regex-constrained sudoers rules. UIDs below 2000 are blocked, only octo_ prefixed usernames are allowed, path traversal is prevented by pattern matching.

# Only high UIDs, octo group, bash shell
# No UID 0, no path traversal, no group escape
Cmnd_Alias OCTO_USERADD = \
  /usr/sbin/useradd -u [2-9][0-9][0-9][0-9] \
  -g octo -s /bin/bash -m -c [a-z0-9_]* octo_[a-z0-9_]*

Container Isolation

Container mode provides full Docker/Podman isolation per session. Each agent runs in its own container with controlled resource limits and network policies.
